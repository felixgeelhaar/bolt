Why bolt vs slog / zerolog / zap

A focused decision tree, not a feature shootout.

You want…Pick
The standard library API, no third-party dependency, no perf budget. log/slog (stdlib)
Zero-alloc speed, slog ergonomics, and first-class OTel trace/span injection. bolt
The zerolog-style chained API with the smallest production diff and a clear migration path. bolt + migration guide
zap's typed-constructor API. bolt's chained API; see migration guide

Bolt is not trying to win a single-digit-nanosecond benchmark shootout. It's the slog handler you can ship into a production Go service without giving up structured perf, OTel correlation, or the slog ergonomics other teams already learned.

Performance

Zero allocations on the hot path; latency varies by hardware. CI gate BenchmarkZeroAllocation requires 0 allocs/op; the benchstat regression workflow catches statistically significant deltas (Mann–Whitney U-test, α=0.05).

Quick start

As a slog.Handler (recommended)

package main

import (
    "log/slog"
    "os"

    "github.com/felixgeelhaar/bolt"
)

func main() {
    logger := slog.New(bolt.NewSlogHandler(os.Stdout, nil))
    slog.SetDefault(logger)
    slog.Info("server starting", "service", "api", "port", 8080)
}

Chained-builder API (hot paths)

package main

import (
    "os"

    "github.com/felixgeelhaar/bolt"
)

func main() {
    log := bolt.New(bolt.NewJSONHandler(os.Stdout))
    log.Info().
        Str("service", "api").
        Int("port", 8080).
        Msg("server starting")
}

Supply-chain hardness

A logging library is imported by every service, so signed artefacts and provenance are not optional.

Per-PR security scan

nox runs on every PR — OSV-aware vulnerability + secret scan, baseline-tracked false positives. Replaces dependabot for security-driven gomod upgrades via the nox-remediate scheduled workflow.

Signed releases

Source archive + SBOM + checksums signed with cosign keyless OIDC (Sigstore). SLSA-3 provenance attestation via slsa-github-generator. Verification commands in SECURITY.md.

Continuous fuzzing

9 Fuzz* targets in fuzz_test.go exercised by the hourly fuzz workflow and a nightly gremlins mutation-testing run. OSS-Fuzz onboarding scaffolding in oss-fuzz/.
